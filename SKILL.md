# A Cheng-Inspired Chinese Narrative

Use this skill to write or revise Chinese prose with the style principles in `style-guide.md`. Do not imitate or reuse protected text from the samples. Apply the method: plain narration, concrete objects, action-led psychology, oral rhythm, restrained feeling, and strict removal of obvious AI-writing habits.

## Workflow

1. Read `style-guide.md` before substantial drafting or revision.
2. Identify the target: polish, rewrite, continue, draft from outline, or diagnose style problems.
3. Preserve the user's facts, plot, names, and intended meaning unless the user asks for freer adaptation.
4. Replace abstract emotion and explanation with visible action, object, dialogue, sound, or bodily detail.
5. Shorten stiff sentences. Prefer spoken Chinese rhythm over essay-like structure.
6. Remove AI-flavor patterns listed in `style-guide.md`: stock transitions, textbook openings, over-structured bullets, slogan endings, vague tool names, fake examples, and forbidden high-frequency phrases.
7. End scenes on an action, a line of dialogue, or a concrete image when possible; avoid summary or uplift.
8. Run the four-layer self-check in `style-guide.md` before final output when the request is about 去除ai味 or style polishing.

## Revision Priorities

- Make the prose specific: food, money, clothes, tools, seats, light, sound, weather, small gestures.
- Let characters reveal themselves through habits, speech, and what they notice.
- Keep the narrator observant but modest. Use uncertainty naturally: 大约, 像是, 似乎, 听说, 后来才知道.
- Keep humor low-key. Let awkwardness and mismatch do the work.
- Keep moral judgment indirect. Do not explain the theme too early.

## Sentence And Voice Rules

- Use short and medium sentences; split long analytical sentences.
- Prefer 就, 倒, 可, 于是, 便, 却, 忽然, 后来, 不料 over formal connectors.
- Avoid inflated phrases such as 时代洪流, 精神图景, 深层隐喻, 人性复杂, 情绪价值.
- Avoid neat three-part structures, slogans, quote-ready aphorisms, and essay conclusions.
- Use dialogue plainly. Prefer “他说” or no tag; avoid explaining every tone or motive.
- For final Chinese prose, avoid colon, em dash, and double quotation marks unless the user explicitly needs them; use 「」 for necessary quoted speech or emphasis.
- Name concrete tools and models. Do not write vague labels like AI工具 or 某个模型 when a specific product name is available.

## Output Rules

- If rewriting, provide the revised Chinese text first.
- If useful, add a brief note listing the main style moves made. If the user asks for QA, use the concise four-layer report format from `style-guide.md`.
- Keep commentary concise; the prose should carry the style.
- For close imitation requests, state that the work is inspired by summarized techniques rather than a direct copy.

# acheng-style-zh

一个用于中文写作润色的 Codex Skill，也可以作为 Claude Code 项目规则使用。

它的目标不是机械模仿阿城，而是提取一种写法，白描、口语、具体物件、动作写心理，少抒情，少解释，少报告腔。尤其适合处理“去除 AI 味”“改得更像人写的”“写得更朴素、更具体”这类任务。

## 适用场景

- 中文叙事、散文、随笔、回忆文本润色
- 去除 AI 写作痕迹
- 把抽象、顺滑、报告式的文字改得更具体
- 写公众号长文、工具体验、产品观察时减少模板味
- 诊断一段文字哪里像 AI 写的

## 文件说明

- `SKILL.md`，Codex Skill 入口文件，负责触发和执行流程
- `style-guide.md`，核心风格指南，包含阿城式叙事规则、反 AI 味禁区和四层自检
- `CLAUDE.md`，Claude Code 项目规则
- `acheng_1.md`、`acheng_2.md`，本地参考样文，用于总结风格，不应直接复制

## Codex 安装

把整个目录放到本机 Codex skills 目录下，目录名建议保持为：

```text
acheng-style-zh
```

Windows 示例：

```text
C:\Users\<你的用户名>\.codex\skills\acheng-style-zh
```

安装后，在新线程里说这些话时，Codex 应该会自动命中：

- 去除ai味
- 去除 AI 味
- 去 AI 痕迹
- 改得像人写的
- 写得更朴素、更具体、更口语
- 按阿城式润色

## Claude Code 使用

Claude Code 不会按 Codex Skill 机制自动发现 `SKILL.md`，但会读取项目里的 `CLAUDE.md`。

把这个仓库作为项目打开，或把 `CLAUDE.md` 和 `style-guide.md` 放进你的写作项目里，就可以让 Claude Code 按同一套规则工作。

## 使用原则

这个 skill 不做句子级仿写，不复制样文中的人物、情节或表达。

它只使用总结出来的方法：

- 用具体物件压住情绪
- 用动作代替心理说明
- 用短句和口语节奏推进
- 避免套话、空泛词、教科书开头
- 写完后按四层自检查一遍

如果只是普通改写，输出改好的正文即可。若用户要求质检，再输出简短的四层检查报告。
